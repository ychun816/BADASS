# P2

## Overview

- configuration and deployment of **VXLAN (RFC 7348)** networks. 
- Based on functional routing stack frm p1, implement a VXLAN network between two hosts encapsulated across a routed backbone.

### Objectives
1. **Static Mode Initialization**: Set up the first VXLAN configuration manually without any dynamic control plane.
2. **Dynamic Multicast Mode**: Evolve the configuration into a dynamic environment using IP multicast groups to dynamically share MAC address information.

### Core Requirements
- **VXLAN Configuration**: Set Virtual Network Identifier (VNI/ID) to `10`. Create interface `vxlan10`.
- **Bridge Setup**: Configure a bridge interface named `br0`.
- **Ethernet Options**: Interface assignments remain customizable across the base OS.
- **Multicast Group**: (Dynamic Mode only) Use a group such as `239.1.1.1` to enable dynamic multicast.
- **Naming Conventions**: Base images and GNS3 equipment names must include the login of one of the team members (e.g., `router_yilin`).
- **Deliverables**: All code must reside in the `/P2` directory. Include exported `.zip` topologies (with portable images) and thoroughly commented configuration files covering both the static and dynamic phases.
